# iReDev 自动化需求开发系统

## 系统简介

iReDev (Intelligent Requirements Development) 是一个面向软件需求工程的自动化需求开发系统。该系统从粗粒度、非结构化的初始自然语言需求描述开始,通过大语言模型驱动的多智能体协作,逐步自动生成高质量的用户需求列表、需求模型以及符合 ISO/IEC/IEEE 29148 标准的软件需求规格说明书(SRS)。

## 核心特性

### 🎯 多智能体协作
- **6个专业智能体**: 访谈型、终端用户、部署者、分析型、文档型、审查型
- **明确角色分工**: 每个智能体模拟真实需求工程中的不同专业角色
- **异步事件驱动**: 通过共享制品池进行协作,而非直接对话

### 🧠 知识驱动
- **5种知识类型**: 领域知识、方法论、标准、模板、策略
- **显式知识注入**: 智能体不依赖模型的隐含知识,而是显式应用专家知识
- **可复用知识库**: 模块化的知识管理系统

### 🔄 多步思考推理
- **链式思考(CoT)引擎**: 多步推理过程,而非直接给出结果
- **可解释性**: 提供中间推导逻辑和推理步骤
- **推理模板**: 针对不同任务类型的专门推理模板

### 📦 共享制品池
- **中心化状态存储**: 所有中间和最终需求制品统一管理
- **版本控制**: 完整的版本历史和变更追踪
- **事件触发**: 制品变更自动触发相关智能体

### 👤 人在环路
- **关键节点审查**: 用户需求列表、需求模型、SRS文档生成后暂停等待审查
- **反馈驱动修订**: 人工反馈写回制品池,驱动智能体重新分析和修订
- **迭代改进**: 支持多轮迭代和修订

### 📋 标准合规
- **ISO/IEC/IEEE 29148**: 符合国际需求工程标准
- **IEEE 830**: 支持IEEE 830 SRS文档结构
- **自动合规检查**: 自动验证文档符合标准要求

## 快速开始

### 安装依赖

```bash
pip install -r requirements.txt
```

### 配置LLM

编辑 `config/agent_config.yaml`:

```yaml
llm:
  type: "openai"  # 或 "claude", "gemini"
  model: "gpt-4"
  api_key: "your-api-key"
```

### 运行完整流程示例

```bash
python examples/complete_workflow_example.py
```

### 使用命令行接口

```bash
# 启动新流程
python run_iReDev.py start --project "我的项目" --domain web

# 查看状态
python run_iReDev.py status --all

# 处理审查
python run_iReDev.py review --list
```

## 系统架构

```
初始需求描述 (粗粒度、非结构化)
    ↓
[访谈型智能体] → 访谈记录
    ↓
[终端用户智能体] → 用户角色、场景、痛点
    ↓
[部署者智能体] → 部署约束、安全需求
    ↓
[分析型智能体] → 系统需求、需求模型
    ↓
[人工审查] ⏸ → 反馈
    ↓
[文档型智能体] → SRS文档
    ↓
[人工审查] ⏸ → 反馈
    ↓
[审查型智能体] → 质量验证
    ↓
最终SRS文档 (符合ISO/IEC/IEEE 29148标准)
```

## 主要组件

### 智能体

- **InterviewerAgent**: 使用5W1H和Socratic提问进行需求访谈
- **EndUserAgent**: 创建用户角色、场景和痛点分析
- **DeployerAgent**: 分析部署约束和安全需求
- **AnalystAgent**: 需求分析和建模
- **ArchivistAgent**: 生成标准SRS文档
- **ReviewerAgent**: 质量验证和审查

### 核心系统

- **RequirementOrchestrator**: 流程编排器
- **ArtifactPool**: 共享制品池
- **EventBus**: 事件总线
- **KnowledgeManager**: 知识管理器
- **HumanReviewManager**: 人在环路管理器
- **ChainOfThoughtEngine**: 链式思考引擎

## 技术特点

1. **过程导向**: 强调需求开发过程的自动化,而非一次性文档生成
2. **知识驱动**: 显式注入专家知识,而非依赖模型隐含知识
3. **可解释性**: 通过CoT推理提供可解释的中间推导逻辑
4. **人在环路**: 关键节点支持人工审查和反馈
5. **事件驱动**: 智能体通过事件异步协作
6. **可扩展性**: 支持多种LLM,易于添加新智能体和知识模块
