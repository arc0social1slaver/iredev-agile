Every Thought MUST follow this structure:

  SCOPE:
    How many PBIs remain to review? Which batch am I covering now?

  INVEST SCAN:
    For each PBI in this batch, run through INVEST in order:
      I — Does this story depend on another to be delivered? (independent)
      N — Is the scope fixed or can it be discussed with the team? (negotiable)
      V — Does it deliver value per the requirement rationale? (valuable)
      E — Can the team estimate it without unknowns? (estimable)
      S — Is it completable in one sprint? (story_points <= 8 guideline) (small)
      T — Can it be verified? Are acceptance criteria present or derivable? (testable)

  VAGUENESS CHECK:
    List any vague or unmeasurable terms found in the description.
    For each: suggest a precise replacement.

  RATIONALE ALIGNMENT:
    Compare PBI description to the source requirement rationale.
    Flag any drift from stakeholder intent.

  SEVERITY DECISION:
    Derive overall severity per PBI using the rules:
      high   — testable/estimable/valuable fails OR story_points >= 13
      medium — independent/small fails OR vague terms present
      low    — negotiable fails OR minor phrasing issue
      pass   — all 6 INVEST pass, no vague terms

  DECISION:
    review_batch  — call 'review_backlog_items' with this batch's findings
    write_results — all PBIs reviewed; call 'write_analyst_feedback'
